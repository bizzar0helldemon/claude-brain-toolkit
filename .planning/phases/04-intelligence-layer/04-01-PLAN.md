<task type="auto">
  <name>Task 1: Create PostToolUse git commit detection hook</name>
  <files>hooks/post-tool-use.sh</files>
  <action>
Create `hooks/post-tool-use.sh` following the established hook pattern (see `hooks/stop.sh` for reference). The script:

1. Reads stdin into HOOK_INPUT via `$(cat)`
2. Sources `~/.claude/hooks/lib/brain-path.sh`
3. Calls `brain_path_validate` — if invalid, `exit 0` (degrade gracefully, do NOT block tool use for brain failures)
4. Extracts `tool_name` and `tool_input.command` from HOOK_INPUT using jq
5. If `tool_name` is not `Bash`, exits 0 (no-op for non-Bash tools)
6. If command does NOT contain `git commit`, exits 0
7. If command contains `--dry-run`, exits 0 (dry runs are not real commits — skip capture)
8. Logs the detection via `brain_log_error "PostToolUse" "git commit detected: $COMMAND"`
9. Constructs a `decision:block` JSON payload with a reason instructing Claude to run `/brain-capture` to preserve patterns from the committed work, then summarize what was committed and captured
10. Emits via `emit_json` and exits 0

The REASON text should be: "A git commit just completed. Before continuing, please run /brain-capture to preserve any useful patterns or decisions from this work session. After capturing, briefly summarize what was committed and what was captured."

IMPORTANT: This hook must NOT use `async: true` in settings.json — `decision:block` requires synchronous execution. Do NOT add a `stop_hook_active` guard — PostToolUse does not loop like Stop hooks do (per research finding).

Make the file executable (`chmod +x`).
  </action>
  <verify>
Run: `bash -n hooks/post-tool-use.sh` (syntax check — must exit 0 with no output).
Run: `echo '{"tool_name":"Bash","tool_input":{"command":"git commit -m \"test\""}}' | BRAIN_PATH=/tmp/test-brain bash hooks/post-tool-use.sh` after `mkdir -p /tmp/test-brain` — should output JSON containing `decision:block`.
Run: `echo '{"tool_name":"Bash","tool_input":{"command":"git status"}}' | BRAIN_PATH=/tmp/test-brain bash hooks/post-tool-use.sh` — should produce no stdout (exit 0 silently).
Run: `echo '{"tool_name":"Bash","tool_input":{"command":"git commit --dry-run"}}' | BRAIN_PATH=/tmp/test-brain bash hooks/post-tool-use.sh` — should produce no stdout (filtered out).
  </verify>
  <done>post-tool-use.sh passes syntax check and correctly detects git commit commands while ignoring non-commit commands and dry runs</done>
