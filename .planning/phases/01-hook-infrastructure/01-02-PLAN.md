**hooks/stop.sh (CRITICAL — loop guard):**
```
#!/usr/bin/env bash
HOOK_INPUT=$(cat)

# CRITICAL: Loop guard MUST be checked BEFORE sourcing anything
# stop_hook_active is true when this hook already fired once this turn
STOP_HOOK_ACTIVE=$(printf '%s' "$HOOK_INPUT" | jq -r '.stop_hook_active // false')
if [ "$STOP_HOOK_ACTIVE" = "true" ]; then
  # Already ran once — let Claude stop. No output, no logging, no sourcing.
  exit 0
fi

source ~/.claude/hooks/lib/brain-path.sh

if ! brain_path_validate; then
  # Brain path invalid — degrade, don't block stop
  exit 0
fi

# Phase 1: scaffold only — future phases add pre-stop capture here
emit_json '{"status":"ok","brain_mode":true}'
exit 0
```

The loop guard check is BEFORE `source brain-path.sh` — this is deliberate. If the lib has any issue, the guard still works. Per user decision: 1 retry allowed (stop_hook_active handles this — first call runs normally, second call exits immediately).

**hooks/post-tool-use-failure.sh:**
```
#!/usr/bin/env bash
HOOK_INPUT=$(cat)
source ~/.claude/hooks/lib/brain-path.sh

if ! brain_path_validate; then
  exit 1
fi

# Log tool failure for post-mortem debugging
TOOL_NAME=$(printf '%s' "$HOOK_INPUT" | jq -r '.tool_name // "unknown"')
ERROR_MSG=$(printf '%s' "$HOOK_INPUT" | jq -r '.error // "no error message"')

brain_log_error "ToolFailure:$TOOL_NAME" "$ERROR_MSG"

# Phase 1: log only — future phases add error pattern matching here
emit_json '{"status":"ok","logged":true,"tool":"'"$TOOL_NAME"'"}'
exit 0
```

This hook is registered with `async: true` in settings.json — it cannot block. It only logs failures for future pattern recognition (Phase 4).

  <action>
Create `statusline.sh` in the project root (distributable — users copy to `~/.claude/statusline.sh`).

```
#!/usr/bin/env bash
input=$(cat)

MODEL=$(printf '%s' "$input" | jq -r '.model.display_name // "Claude"')
AGENT=$(printf '%s' "$input" | jq -r '.agent.name // ""')
PCT=$(printf '%s' "$input" | jq -r '.context_window.used_percentage // 0' | cut -d. -f1)

if [ "$AGENT" = "brain-mode" ]; then
  printf '\360\237\247\240 [%s] %s%%\n' "$MODEL" "$PCT"
else
  printf '[%s] %s%%\n' "$MODEL" "$PCT"
fi
```

Key details:
- Brain emoji (U+1F9E0) is printed as UTF-8 octal `\360\237\247\240` via printf for bash/zsh compatibility
- `agent.name` is only present when launched with `--agent brain-mode` — use `// ""` fallback for absent field
- `used_percentage` may be a float — `cut -d. -f1` truncates to integer
- Shows brain emoji + model name + context % when in brain mode
- Shows just model name + context % otherwise (Claude's discretion: simple, useful baseline)
