<task type="auto">
  <name>Task 1: Create BRAIN_PATH validation library</name>
  <files>hooks/lib/brain-path.sh</files>
  <action>
Create `hooks/lib/brain-path.sh` as a sourced library (no shebang — it is sourced, not executed directly). This file provides three functions used by every hook:

**brain_path_validate():**
- Check if `$BRAIN_PATH` is set (non-empty). If unset:
  - stderr: contextual explanation — what BRAIN_PATH is, why it matters, exact fix command (`export BRAIN_PATH="/path/to/your/vault"` in shell profile). Self-contained, no forward references to commands that don't exist yet. Per user decision: explanation style, not a one-liner.
  - stdout: JSON with `"error"` and `"degraded":true` fields for Claude to read. Per user decision: dual-channel errors.
  - return 1
- Check if `$BRAIN_PATH` directory exists (`[ -d "$BRAIN_PATH" ]`). If not:
  - stderr: explain the directory doesn't exist, suggest `mkdir -p "$BRAIN_PATH"`, mention updating BRAIN_PATH if vault moved. Per user decision: offer to create (mediated by Claude reading JSON, not interactive prompt — hooks cannot prompt).
  - stdout: JSON with `"error"` field including the path, `"degraded":true`, and `"offer_create":true` so Claude can surface the offer.
  - return 1
- If valid, return 0 with no output.

**brain_log_error(event, message):**
- UTC timestamp via `date -u +"%Y-%m-%dT%H:%M:%SZ"`
- Append `[timestamp] event: message` to `$BRAIN_PATH/.brain-errors.log`
- Only write if `$BRAIN_PATH` is a valid directory (guard against logging when path is invalid)
- Per user decision: persistent error log for post-mortem debugging.

**emit_json(json_string):**
- Validate the JSON string with `printf '%s' "$json" | jq empty >/dev/null 2>&1`
- If valid: print to stdout with `printf '%s\n' "$json"`
- If invalid: write error to stderr, call `brain_log_error "JSONValidation" "..."`, exit 0 (don't break the session)
- Per user decision: self-validate before returning, catches corruption early.

Use `printf` not `echo` for all output (portable across bash/zsh). All functions must work in bash 3.2+ and zsh 5.0+.

After creating the file, run `chmod +x hooks/lib/brain-path.sh` (even though it's sourced, not executed — belt and suspenders).
  </action>

<task type="auto">
  <name>Task 2: Create settings.json with hook registration and statusline</name>
  <files>settings.json</files>
  <action>
Create `settings.json` in the project root. This is the distributable template that users will copy to `~/.claude/settings.json`. It registers all 4 lifecycle hooks and the statusline.

Structure (follow the exact Claude Code hooks configuration format from research):

```json
{
  "env": {
    "BRAIN_PATH": ""
  },
  "statusLine": {
    "type": "command",
    "command": "~/.claude/statusline.sh"
  },
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/session-start.sh",
            "timeout": 10
          }
        ]
      }
    ],
    "PreCompact": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/pre-compact.sh",
            "timeout": 10
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/stop.sh",
            "timeout": 10
          }
        ]
      }
    ],
    "PostToolUseFailure": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/post-tool-use-failure.sh",
            "timeout": 10,
            "async": true
          }
        ]
      }
    ]
  }
}
```

Key details:
- `env.BRAIN_PATH` is empty string — user must set it. This ensures BRAIN_PATH is injected into hook subprocesses (research pitfall 6: shell profile alone is not enough for non-interactive subshells).
- PostToolUseFailure has `async: true` — it cannot block (research pitfall 7), and async means it never slows the session.
- All timeouts are 10 seconds — hooks in Phase 1 are lightweight validation only.
- Leave BRAIN_PATH value empty — the onboarding phase (Phase 3) will guide the user to set it. Do NOT hardcode a path.

Also create `hooks/` and `hooks/lib/` directories if they don't exist (mkdir -p).
  </action>

  </verify>
