<task type="auto">
  <name>Task 1: Create brain-context.sh vault query and budget library</name>
  <files>hooks/lib/brain-context.sh</files>
  <action>
Create `hooks/lib/brain-context.sh` as a sourced library (same pattern as brain-path.sh — no shebang, comment header). This library provides all functions needed for vault context assembly. It sources brain-path.sh for access to brain_log_error and emit_json.

**Functions to implement:**

1. `get_frontmatter_field "$field" "$file"` — Extract a simple `key: value` YAML frontmatter field from a markdown file. Use the awk-based approach from research (handles first `---` as opening delimiter): `awk '/^---/{found++; next} found==1{print} found==2{exit}' "$file" | grep "^${field}:" | head -1 | sed "s/^${field}:[[:space:]]*//" | tr -d '"'"'"`. Strip `\r` for Windows compatibility.

2. `get_mtime "$file"` — Get file modification time as Unix timestamp. Try GNU `stat -c '%Y'` first, then BSD `stat -f '%m'`, then fallback to `date -r "$file" +%s`. If all fail, return 0 (treat as oldest). Log warning via brain_log_error on all-fail.

3. `get_project_name "$cwd"` — Get canonical project name. Use `git -C "$cwd" rev-parse --show-toplevel 2>/dev/null` to find repo root, then `basename` it. Also extract git remote repo name as secondary candidate via `git -C "$cwd" remote get-url origin 2>/dev/null | sed 's/.*[:/]\([^/]*\)\.git$/\1/' | sed 's/.*[:/]\([^/]*\)$/\1/'`. Fall back to `basename "$cwd"` if not in a git repo. Print space-separated candidates to stdout.

4. `count_tokens "$text"` — Count tokens. If `ttok` is available (`command -v ttok`), pipe text through it. If ttok fails or is unavailable, fall back to char heuristic (`wc -c | awk '{print int($1/4)}'`). Log a warning to brain_log_error on first fallback use (use a module-level variable `_TTOK_WARNED` to warn only once per invocation).

5. `entry_matches_project "$file" "$project_candidates"` — Check if a vault file's frontmatter `project:` field matches any of the space-separated project candidates. Return 0 (match) or 1 (no match). Empty project field = no match (return 1).

6. `collect_vault_entries "$cwd"` — Walk `$BRAIN_PATH/` recursively for `.md` files. For each, check `entry_matches_project`. Collect matches into two categories: project-specific entries and global entries (entries with `type: preference` or `type: global` in frontmatter, or files in a `preferences/` subdirectory). Sort project entries by mtime (most recent first). Print file paths to stdout, one per line, project entries first then global entries.

7. `load_brain_md "$cwd"` — Check for `.brain.md` in the project root (use git repo root if available, else cwd). If exists, read its content. Return content via stdout. Separate allocation: `.brain.md` gets up to 500 tokens outside the main vault budget. If it exceeds 500 tokens, truncate and append `\n[.brain.md truncated — 500 token limit]`. Log truncation warning.

8. `build_brain_context "$cwd" "$source"` — Main entry point. Calls collect_vault_entries, then accumulates entry content in priority order within token budget (`${BRAIN_TOKEN_BUDGET:-2000}`). For `source:"compact"`, call `is_entry_new_or_changed` to only include delta entries. For `source:"startup"`, `source:"resume"`, `source:"clear"`, do full load. Prepend `.brain.md` content (from load_brain_md) outside the vault budget. Print assembled context to stdout.

9. `build_summary_block "$cwd" "$source" "$project_count" "$pitfall_count" "$global_active" "$newest_date"` — Build the summary block in the approved format:
    ```
    Brain loaded for {project-name}
       {N} project notes (newest: {date})
       {N} pitfalls
       Global preferences active
    ```
    Use the brain emoji. Functional/minimal tone. If first-time project (zero project entries, no .brain.md): append line `No project notes yet — ask me to scan this project for brain cataloging`. Print to stdout.

10. `write_session_state "$project" "${loaded_files[@]}"` — Write `$BRAIN_PATH/.brain-session-state.json` atomically (temp file + mv). JSON structure: `{"project":"...","loaded_at":"ISO8601","entries":[{"path":"...","mtime":"..."},...]}`. Use `jq -n --arg` for safe construction.

11. `is_entry_new_or_changed "$file"` — Read `.brain-session-state.json`, check if file's path exists and mtime matches. Return 0 if new/changed, 1 if unchanged. Missing/corrupt state file = treat everything as new.

12. `track_loaded_entries` — Module-level arrays `_LOADED_FILES=()` and counters `_PROJECT_COUNT=0`, `_PITFALL_COUNT=0`, `_GLOBAL_ACTIVE=false`, `_NEWEST_MTIME=0`. Updated by build_brain_context as it accumulates entries. Used by the caller to build the summary block.

**Important implementation details:**
- Use `printf` throughout (not echo) for portability per Phase 1 convention.
- All JSON construction uses `jq -n --arg` or `jq -Rs .` — never string concatenation with user content.
- `chmod +x` not needed (sourced library), but add it as belt-and-suspenders per Phase 1 pattern.
- Pitfall detection: count entries where frontmatter contains `type: pitfall` or `tags:` containing `pitfall`.
- For collecting vault entries, skip hidden files/dirs (starting with `.`) and the `.brain-session-state.json` / `.brain-errors.log` files.

<task type="auto">
  <name>Task 2: Update session-start.sh with full vault context injection</name>
  <files>hooks/session-start.sh</files>
  <action>
Replace the Phase 1 scaffold in `hooks/session-start.sh` with the full Phase 2 implementation. The hook now:

1. Reads HOOK_INPUT from stdin (already done in scaffold).
2. Sources `brain-path.sh` and validates (already done). On failure, exit 1 (existing behavior).
3. Sources `brain-context.sh` (new).
4. Parses `source` and `cwd` from HOOK_INPUT JSON via jq.
5. Calls `build_brain_context "$CWD" "$SOURCE"` to get the assembled vault content within budget.
6. Calls `build_summary_block` with the tracked counters to get the summary block.
7. Combines summary block + vault context into ADDITIONAL_CONTEXT string.
8. Builds the final JSON using `jq -n --arg ctx "$ADDITIONAL_CONTEXT" '{"hookSpecificOutput":{"hookEventName":"SessionStart","additionalContext":$ctx}}'` — NEVER string concatenation.
9. Emits via `emit_json`.
10. Calls `write_session_state` with the project name and loaded file paths.
11. Writes `BRAIN_LOADED=1` to `$CLAUDE_ENV_FILE` if that var is set (so downstream hooks know brain loaded).
12. Logs to brain_log_error: "SessionStart" with source and number of entries loaded.
13. Exits 0.

**Source-aware behavior (per research Pattern 6):**
- `source:"startup"` / `source:"resume"` / `source:"clear"` — full load
- `source:"compact"` — delta load (only new/changed entries since last session state)

**Edge cases:**
- Empty vault (no entries match): Still inject global prefs. If no global prefs either, inject summary block only with the "no project notes" message and the offer to scan.
- CWD empty or missing from hook input: Fall back to current working directory via `pwd`.
- Large `.brain.md`: Handled by load_brain_md's 500-token cap in brain-context.sh.

**What NOT to change:** Keep the `#!/usr/bin/env bash` shebang. Keep the `HOOK_INPUT=$(cat)` on line 1. Keep the brain-path.sh source and validation. Replace everything after the validation block.

Reference the research patterns (Pattern 1 and Pattern 6) for the exact JSON structure. The key insight from research: `additionalContext` value must be a JSON string — `jq -n --arg` handles all escaping.
