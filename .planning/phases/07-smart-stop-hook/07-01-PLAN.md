  <name>Task 1: Add transcript signal detection to stop.sh</name>
  <files>hooks/stop.sh</files>
  <action>
Modify hooks/stop.sh in-place. Keep the existing structure (loop guard first, then source, then brain_path_validate) but add signal detection between validation and the block emission.

The modified script should follow this exact flow:

1. Read HOOK_INPUT (existing — keep as-is)
2. Loop guard check on stop_hook_active (existing — keep as-is, do NOT move)
3. Source brain-path.sh (existing — keep as-is)
4. brain_path_validate (existing — keep as-is)
5. NEW: Extract transcript_path from HOOK_INPUT using jq
6. NEW: Default TOOL_COUNT=0, HAS_GIT_COMMIT=0, HAS_FILE_CHANGES=0
7. NEW: If transcript_path is non-empty AND file exists, parse it:
   - TOOL_COUNT: count tool_use entries in assistant messages using `jq -r 'select(.type == "assistant") | .message.content[]? | select(.type == "tool_use") | .name' | wc -l | tr -d ' '`
   - HAS_GIT_COMMIT: check Bash tool commands for "git commit" using `grep -c 'git commit' || echo 0`
   - HAS_FILE_CHANGES: count Write/Edit tool calls using `select(.name == "Write" or .name == "Edit") | .name | wc -l | tr -d ' '`
8. NEW: If all three signals are 0, exit 0 silently (no output, no logging)
9. MODIFIED: Log with signal counts: "Capture trigger fired (tools: $TOOL_COUNT, commits: $HAS_GIT_COMMIT, files: $HAS_FILE_CHANGES)"
10. Emit decision:block with existing REASON text (keep existing — do NOT change the capture prompt wording)
11. exit 0

Note on STOP-01 (error resolution detection): Error resolutions are captured implicitly by TOOL_COUNT > 0. If errors were being resolved, tool calls were being made. A separate HAS_ERROR_RESOLUTION signal is not worth the complexity for v1.1 (per 07-RESEARCH.md Open Question 2 recommendation). Any session where Claude resolved errors will have TOOL_COUNT > 0 and will trigger capture.

Critical constraints:
- Do NOT move the stop_hook_active guard — it MUST stay before any source statement
- Do NOT change the REASON text or emit_json pattern — keep existing capture prompt
- Do NOT use exit 2 — use exit 0 with decision:block (per existing pattern)
- Use `2>/dev/null` on all jq calls to suppress errors on malformed lines
- Use `tr -d ' '` after wc -l to handle whitespace
- Use `|| echo 0` after grep -c to handle zero-match exit code
- Guard transcript reading with `[ -n "$TRANSCRIPT_PATH" ] && [ -f "$TRANSCRIPT_PATH" ]`

Reference the "Full Detection Logic Skeleton" in 07-RESEARCH.md — the implementation should match that structure closely.
