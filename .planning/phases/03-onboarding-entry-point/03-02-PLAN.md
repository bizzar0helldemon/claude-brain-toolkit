**Phase 6: Update ~/.claude/settings.json**
This is the most critical step. Must MERGE brain hooks and statusline into the existing settings.json WITHOUT destroying existing entries (like the gsd-check-update.js SessionStart hook).

Use jq to merge. The approach:
1. Read existing settings.json
2. Add/merge brain hooks into each hook type (SessionStart, PreCompact, Stop, PostToolUseFailure)
3. Add/update statusLine to the brain statusline
4. Write back atomically (tmp + mv)

The merge must be IDEMPOTENT — running setup.sh twice should not duplicate hook entries. Use a strategy: check if the brain hook command string already exists before adding.

Concrete jq approach for merging hooks:
```bash
SETTINGS="$HOME/.claude/settings.json"
TEMP="$SETTINGS.tmp"

# Build the brain hooks object
BRAIN_HOOKS=$(cat <<'HOOKS_EOF'
{
  "SessionStart": [{"hooks":[{"type":"command","command":"~/.claude/hooks/session-start.sh","timeout":10}]}],
  "PreCompact": [{"hooks":[{"type":"command","command":"~/.claude/hooks/pre-compact.sh","timeout":10}]}],
  "Stop": [{"hooks":[{"type":"command","command":"~/.claude/hooks/stop.sh","timeout":10}]}],
  "PostToolUseFailure": [{"hooks":[{"type":"command","command":"~/.claude/hooks/post-tool-use-failure.sh","timeout":10,"async":true}]}]
}
HOOKS_EOF
)

# Merge: for each hook type, append brain hook if not already present (check by command string)
# Then set statusLine
jq --argjson bh "$BRAIN_HOOKS" '
  # For each brain hook type, add to existing hooks array if command not already registered
  reduce ($bh | keys[]) as $key (.;
    if (.hooks[$key] // [] | map(.hooks[]?.command) | index($bh[$key][0].hooks[0].command))
    then .
    else .hooks[$key] = ((.hooks[$key] // []) + $bh[$key])
    end
  )
  | .statusLine = {"type":"command","command":"~/.claude/statusline.sh"}
' "$SETTINGS" > "$TEMP" && mv "$TEMP" "$SETTINGS"
```
