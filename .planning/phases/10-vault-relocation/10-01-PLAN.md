  <name>Task 1: Create /brain-relocate slash command</name>
  <files>commands/brain-relocate.md</files>
  <action>
Create `commands/brain-relocate.md` following the exact format of `commands/brain-add-pattern.md` (frontmatter with name/description, then step-by-step instructions for Claude to follow).

The command must implement these steps in order:

**Step 1 — Determine current state:**
- Read current BRAIN_PATH from env (`echo "$BRAIN_PATH"`) and from settings.json (`jq -r '.env.BRAIN_PATH' ~/.claude/settings.json`)
- Tell the user their current vault path
- If BRAIN_PATH is not set in either location, tell the user to run `/brain-setup` instead and stop

**Step 2 — Get the new path:**
- If `$ARGUMENTS` is provided, use it as the new path
- Otherwise ask: "Where is the new vault location?"

**Step 3 — Validate the new path:**
- Expand the path (handle ~ and $HOME): `NEW_PATH=$(eval echo "<user-provided-path>")`
- Check directory exists: `[ -d "$NEW_PATH" ]`
- If it does not exist, ask whether to create it or if they meant a different path
- If creating: `mkdir -p "$NEW_PATH"`

**Step 4 — Optionally copy vault contents:**
- Only offer this if the OLD path exists AND the NEW path is empty (no files)
- If user wants to copy: `cp -r "$OLD_PATH"/. "$NEW_PATH"/` (copies hidden files too)
- After copy, check for broken symlinks: `find "$NEW_PATH" -type l ! -exec test -e {} \; -print` — warn if any found (informational, not blocking)

**Step 5 — Update settings.json (jq atomic write):**
```bash
SETTINGS="$HOME/.claude/settings.json"
if [ ! -f "$SETTINGS" ]; then
  printf '{"env":{}}\n' > "$SETTINGS"
fi
jq --arg p "$NEW_PATH" '.env.BRAIN_PATH = $p' "$SETTINGS" > "$SETTINGS.tmp" && mv "$SETTINGS.tmp" "$SETTINGS"
```

**Step 6 — Update shell profile (portable sed, NO sed -i):**
- Detect profile: check ~/.zshrc, ~/.bash_profile, ~/.bashrc in order
- If `grep -q 'export BRAIN_PATH=' "$PROFILE"` matches, replace with sed temp+mv:
  `sed 's|^export BRAIN_PATH=.*|export BRAIN_PATH="'"$NEW_PATH"'"|' "$PROFILE" > "$PROFILE.tmp" && mv "$PROFILE.tmp" "$PROFILE"`
- If no match, append:
  `printf '\n# Claude Brain vault path\nexport BRAIN_PATH="%s"\n' "$NEW_PATH" >> "$PROFILE"`
- If a non-matching BRAIN_PATH line exists (e.g., without `export` keyword), warn the user about the existing line

**Step 7 — Post-relocate verification:**
- Read back from settings.json: `jq -r '.env.BRAIN_PATH' ~/.claude/settings.json` — must match NEW_PATH
- Check directory exists and is readable: `[ -d "$NEW_PATH" ]`
- Check vault structure: `[ -d "$NEW_PATH/brain-mode" ] || [ -f "$NEW_PATH/.brain-state" ]` — warn (not fail) if empty/new vault

**Step 8 — Confirm and instruct restart:**
- Report success with checklist: settings.json updated, shell profile updated, vault verified
- Tell user which profile file was updated
- MUST tell user: "Restart any open Claude Code sessions for the new vault path to take effect in hooks."

Important constraints:
- Always double-quote "$NEW_PATH" and "$OLD_PATH" in every command (spaces in paths)
- Never use `sed -i` (GNU vs BSD incompatibility) — always use temp file + mv
- Never use `mv` to physically move the vault — use `cp -r` for safety
- The command is Claude-orchestrated (Claude reads these instructions and executes via Bash/Read/Write tools) — there is no standalone shell script
