# Requirements: Claude Brain Mode

**Defined:** 2026-03-19
**Core Value:** The brain compounds over time — every session makes future sessions smarter by actively capturing and applying knowledge without the user having to ask.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Hook Infrastructure

- [x] **HOOK-01**: Stop hook loop guard baked into all brain mode hook templates
- [x] **HOOK-02**: Exit code discipline enforced — `exit 2` for blocking hooks, `exit 1` for non-blocking
- [x] **HOOK-03**: BRAIN_PATH validation on every hook invocation with clear error if missing/invalid
- [x] **HOOK-04**: JSON output self-test included in hook scaffolding template

### Session Lifecycle

- [x] **LIFE-01**: SessionStart hook loads relevant vault context for current project (pitfalls, patterns, preferences)
- [x] **LIFE-02**: Pre-clear auto-capture triggers `/brain-capture` then `/daily-note` before `/clear`
- [x] **LIFE-03**: Token budget management limits vault content injection to prevent context window bloat
- [x] **LIFE-04**: Milestone auto-capture triggers after commits, merges, and phase completions
- [x] **LIFE-05**: Error pattern recognition detects recurring errors and surfaces past solutions from vault

### Statusline

- [x] **STAT-01**: Brain emoji appears in statusline left of model name when brain mode is active

### Onboarding

- [x] **ONBR-01**: First-run guided setup walks user through creating vault directory and configuring BRAIN_PATH
- [x] **ONBR-02**: `claude --agent brain-mode` entry point launches brain mode via native subagent pattern

## v2 Requirements

### Statusline

- **STAT-02**: Color-coded brain states in statusline (idle vs active processing)

### Session Lifecycle

- **LIFE-06**: Idle detection offers to summarize or capture current state when user pauses

### Onboarding

- **ONBR-03**: Vault relocate command updates BRAIN_PATH if vault directory moves

### Adaptive Mentoring

- **MENT-01**: Pattern encounter tracking records frequency per pattern/pitfall
- **MENT-02**: Progressive response changes behavior based on encounter count (warn -> silent fix -> investigate)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Multi-vault support | One brain per system keeps things simple |
| Cloud sync | Vault is local filesystem only — no external services |
| Non-Claude-Code integrations | This is specifically for the Claude Code CLI |
| New brain skills beyond orchestration | Will emerge from usage, not pre-designed |
| Mobile/web interface | CLI-only tool |
